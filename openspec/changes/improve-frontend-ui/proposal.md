## Why

当前 `WebJson` 的前端界面功能完整但视觉和交互体验较为朴素，结果展示区可读性、错误提示、移动端适配与可访问性仍有改进空间。提升前端 UI/UX 能让用户更容易地粘贴/加载 JSON、理解解析错误并更方便地下载结果，从而提高整体可用性和满意度。

## What Changes

- 改进页面布局与样式，使界面在桌面和移动端均有良好显示（响应式设计）。
- 优化结果展示区（可切换原始/美化/压缩视图，增强可读性，调整配色与字体、增加可选择的缩进选项）。
- 增强错误展示：将后端解析错误以高亮/位置提示或可复制的错误信息呈现给用户。
- 改进文件交互：更清晰的文件选择与读取反馈、显示文件名/大小、对大文件给出提示。
- 增加无服务器端下载（客户端 Blob 下载）与可选的“保存为”文件名输入。
- 可选：增加浅色/深色主题切换和基本无障碍支持（键盘导航、语义化标签）。

**Non-goals**

- 不在本次变更中实现后端持久化存储或服务器端文件保存接口（仍以前端 Blob 下载为主）。
- 不进行完整的视觉重设计或替换前端框架（继续使用原生 HTML/vanilla JS）。

## Capabilities

### New Capabilities
- `frontend-ux`: 改进前端用户体验，包括响应式布局、改进的结果展示、错误高亮、文件交互与下载体验。

### Modified Capabilities
- 无（本次变更仅改进前端实现与体验，不改变后端 API 行为或接口契约）。

## Impact

- 受影响文件/模块：
  - `public/index.html` — 结构与无障碍改进
  - `public/style.css` — 响应式样式、主题样式
  - `public/app.js` — 文件读取反馈、错误显示增强、下载逻辑、视图切换
  - 文档：更新 `README.md` 与 `docs/PROJECT_SPEC.md` 中的前端使用说明
- API 影响：无。`POST /format` 行为与返回契约保持不变（仍返回 200 成功和 400 错误 JSON）。
- 依赖：不新增后端依赖；前端仍使用原生 API，无需额外库。

## Success Criteria

- 页面在常见桌面浏览器和移动浏览器上布局正常（窗口宽度从 320px 到 1440px）。
- 用户能通过粘贴或选择文件获得即时反馈并成功格式化/压缩 JSON。
- 错误场景下用户能清晰看到解析错误信息并能定位问题（错误消息可复制）。
- 下载功能在主流浏览器（Chrome/Firefox/Edge/Safari）中可用并能保存为 `.json` 文件。
- 无明显回归：后端接口不被修改，现有自动化/手工测试通过。
